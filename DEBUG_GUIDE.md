# 텔레그램 인증 문제 해결 가이드

## 🔍 문제 진단

### 1. "Requesting..."에서 멈춤 현상

이 문제는 다음과 같은 원인으로 발생할 수 있습니다:

#### **A. 서버 연결 문제**
- Render Web Service가 실행되지 않음
- 서버가 크래시됨
- 네트워크 연결 문제

#### **B. API 자격 증명 문제**
- 잘못된 API ID
- 잘못된 API Hash
- 텔레그램 개발자 계정 문제

#### **C. 전화번호 형식 문제**
- +82 형식이 아님
- 잘못된 전화번호

## 🛠️ 해결 방법

### 1. 서버 상태 확인

브라우저에서 다음 URL을 확인하세요:
```
https://your-app-name.onrender.com/health
```

정상 응답 예시:
```json
{
  "status": "healthy",
  "timestamp": "2024-01-01T00:00:00.000Z",
  "uptime": 123.45,
  "memory": {...}
}
```

### 2. API 자격 증명 확인

1. **텔레그램 개발자 계정 접속**
   - https://my.telegram.org/ 접속
   - 본인 계정으로 로그인

2. **API 정보 확인**
   - "API development tools" 클릭
   - API ID (숫자)
   - API Hash (32자리 문자열)

3. **올바른 형식**
   ```
   API ID: 12345678 (숫자만)
   API Hash: abcdef1234567890abcdef1234567890 (32자리)
   ```

### 3. 전화번호 형식 확인

올바른 형식:
```
+821012345678
+82101234567
```

잘못된 형식:
```
01012345678 (앞에 +82 없음)
821012345678 (앞에 + 없음)
```

### 4. 브라우저 콘솔 확인

F12를 눌러 개발자 도구를 열고 Console 탭에서 에러 메시지를 확인하세요.

## 🚨 일반적인 에러 메시지

### "서버에 연결할 수 없습니다"
- Render Web Service가 실행되지 않음
- 서버가 크래시됨

### "API ID가 올바르지 않습니다"
- API ID가 숫자가 아님
- 잘못된 API ID

### "API Hash가 올바르지 않습니다"
- API Hash가 32자리가 아님
- 잘못된 API Hash

### "전화번호 형식이 올바르지 않습니다"
- +82로 시작하지 않음
- 숫자가 아님

## 📱 텔레그램 앱에서 인증코드 확인

1. **텔레그램 앱 열기**
2. **설정** → **개인정보 및 보안**
3. **활성 세션** 확인
4. **새로운 로그인** 알림 확인

## 🔧 추가 디버깅

### 서버 로그 확인 (Render Dashboard)
1. Render Dashboard 접속
2. 해당 서비스 클릭
3. "Logs" 탭에서 실시간 로그 확인

### 로컬 테스트
```bash
# 로컬에서 서버 실행
npm install
npm start

# 브라우저에서 http://localhost:3000 접속
```

1. 브라우저 콘솔 에러 메시지
2. Render 서버 로그
3. 사용한 API ID (숫자만)
4. 전화번호 형식 (마스킹 처리)
